1.计网概述

计算机网络是这个时代最伟大的设计之一，人类发展的里程碑之一

一些宏观的描述，课程的目的和要求

2.直连网络

2.1 数据通信基本概念

信息：语音、文字、图像、视频等，通信的目的即传送信息

数据：运送消息的载体，有意义的符号序列

信号：数据的电气的或电磁的表现，以光速传播的电磁波（又分为模拟信号和数字信号）

码元：在使用时域波形表示数字信号时，代表不同离散数值的基本波形

单工通信、半双工通信、全双工通信的含义

基带信号：来自信源的信号，如电话语音信号、计算机输出的代表各种文字或图像文件的数据信号

调制：将基带信号变换方式后进行传输，目的如下：

  • 将信号变换为便于传送的形式
  • 提高性能，特别是抗干扰能力
  • 有效利用频带

调制的方法：调幅（AM）、调频（FM）和调相（PM）

香农公式：$C = Wlog_2(1 + S / N)$, 其中C –信道容量（bps），W–信道带宽（HZ），S–信号平均功率；N – 高斯白噪声功率，$S/N$–信噪比，常表示为SNR（db）：$SNR = 10log_10(S/N)$

2.2 计算机网络组成及体系结构模型

点对点链路：链路的一端的单个发送发和另一端的单个接收方（1v1）

广播链路：能够让多个结点都连接到相同的、单一的、共享的广播信道（1v多，谁都可以当那个1）

网络体系结构

网络并不是一成不变的，必须适应所依赖的底层技术的变化和上层应用对网络需求的变化

建立网络体系结构，来指导网络的设计和实现

  • 能够抽象地讨论和研究网络技术
    • 抽象，隐藏定义良好的接口背后的细节，是系统设计者用于处理复杂性的基本工具
    • 具体是指，定义一个能够捕获系统重要性征的模型，并将这个模型封装为一个对象，为系统的其它组件提供一个可操作的接口，同时向对象的使用者隐藏对象的实现细节
  • 使网络的实现与网络的功能扩展能够相互独立地各自发展

模块化：模块化的协议栈和良好定义的模块接口

  • 优点：独立性强，功能简单，模块易于实现
  • 缺点：适应性差，难以维护，对系统实现是场灾难

两层结构：应用层和物理层，两层的每种类型一一适配

  • 优点：相比于模块化，结构更清晰，更易于维护
  • 缺点：每有一个新的应用，都需要和物理层的每一个类型去适配

三层结构：中间层实现对物理层的抽象，为应用层提供一个统一的接口

  • 优点：极大降低了适配的工作量，$M*N>>M + N$
  • 缺点：引入适配层增加了设计难度

分层的优点

  • 各层相互独立：将建造一个网络的问题分解成多个可处理的部分，一层解决一个问题
  • 灵活性好：任何层发生变化时，只要接口不变，上下层不受影响
  • 结构上可以分割开，各层都可以采用最合适的技术实现
  • 易于实现和维护
  • 能促进标准化工作

理论模式是：OSI七层模型，实际模型是：TCP/IP体系结构

2.3 直连网络构建的关键技术

奇偶校验：差错往往以突发的方式聚集在一起，并非独立发生，校验的效果一般

循环冗余校验：信息M末尾先加上k个0得到扩展信息T，S = T % C，最后的p = T - S，接收方验证P % C == 0 ？

两种ARQ算法

停等算法：发送方传输一帧后，在下一帧传送前等待ACK信息；如果在规定时间内没收到自动重发（链路上只有一个未确认的数据帧，远低于链路的容量）

时延带宽积：（以比特位单位的链路长度）表示该链路能容纳的比特数

滑动窗口算法：可靠传输、高效传输、按需传送、流量控制

  • 保持管道满载，提升传输速率
  • 并发传送数据
  • 每一个数据帧赋予一个序列号

媒体共享技术：频分复用、时分复用、统计时分复用、波分复用、码分复用

动态媒体接入控制

  • ALOHA
  • 时隙ALOHA
  • 载波侦听多点接入（CSMA）
  • 带碰撞检测（CSMA/CD）
  • 嗲碰撞避免（CSMA/CA）

3. 交换网络与网络互联

3.1 交换网络

设计目标

  • 数据只朝向目的结点方向传送（转发，forward），引入交换节点
  • 转发规则是网络自己学习生成的，不需要外界参与

网络中存在冗余链路

  • 网络拓扑由树状结构变成图状结构
  • 数据转发过程中，形成环路，造成广播风暴

解决办法

  • 为网络中每对（源–目的）结点分配唯一确定的一条路径
  • 这些路径构成了一棵树（生成树，Spanning Tree）

生成树协议

  • 选一个根节点，其它每个节点计算确定到根节点的最短路径
  • 保证是最小生成树（Minimum Spanning Tree）

3.2 网络互联

IP设计思路：向上提供最基本的、简单的、灵活的数据报传输服务

  • 无连接
    • 网络发送分组时不需要先在源和目的结点间建立连接
    • 每一个分组（IP数据报）独立发送，不进行编号，与其前后的分组无关
  • 尽最大努力交付
    • 网络层不提供服务质量的承诺
    • 传送的分组可能出错、丢失、重复和乱序，也不保证分组传送的时限
    • 如果主机（即端系统）中的进程之间的通信需要可靠传输，由主机中的传输层负责（包括差错处理、流量控制等），与网络节点（路由器）无关

优点：

  • 协议设计简单
  • 中间转发设备功能简单，成本低

网际协议IP

  • TCP/IP体系中最主要的协议之一
  • 编制规则、数据报格式、分组处理规则

支撑协议

  • 路由选择协议：路径选择
  • 地址解析协议ARP：实现IP地址与硬件地址的映射
  • 网际控制报文协议ICMP：网络控制与诊断
  • 网际组管理协议IGMP：IP多播

分类IP地址：IP地址 = 网络号 + 主机号

  • A类、B类、C类地址为单播地址
  • D类为组播地址
  • E类为保留地址以备特殊用途

分类IP地址问题

问题一：IP地址的利用率低，网络号和主机号位数固定，导致网络规模不可调，造成分配的不合理

  • 各类网络包含的主机数固定
  • 一个A类网络可包含的主机数：$2^{24} – 2= 16,777,214$
  • 一个B类网络可包含的主机数：$2^{16} – 2= 65534$
  • 一个C类网络可包含的主机数：$2^8 – 2= 254$
  • “无法根据网络规模灵活选择网络地址
  • 4个结点的网络使用一个完整的C类地址，浪费252个IP地址
  • 1个10BaseT以太网最大结点数1024，得用一个B类地址，浪费6万多个IP地址
  • 分配的是网络号而不是40亿个独立的IP地址，耗尽的速度快
  • 126个 (27 – 2)A类网络号分配完，就用尽了一半的IP地址，20亿
  • 16,383个 (214 – 2)B类网络号分配完，就用尽了1/4的IP地址，10亿

问题二：每个物理网络分配一个网络号使得路由表过大

  • 为每一个网络号增加一条目
  • 大转发表增加路由器开销，增大路由查找延迟，减低路由器性能

问题二：每个物理网络分配一个网络号使得路由表过大

  • 为每一个网络号增加一条目
  • 大转发表增加路由器开销，增大路由查找延迟，减低路由器性能

划分子网：两级的 IP 地址变成为三级–IP 地址中又增加了一个“子网号字段”，从主机号借用若干个位作为子网号 subnet-id，不改变原来的网络号net-id，”IP地址 = 网络号+子网号+主机号

无分类域间路由 CIDR：打破地址分类的严格界线，网络号为可变长—更有效地分配 IPv4 地址空间，仅仅使用转发表中的一条记录来知道如何到达多个不同网络 —- 路由聚合，IP地址 = 网络前缀(network prefix) + 主机号

网络地址转换NAT：负责私有地址与全局地址之间的翻译
