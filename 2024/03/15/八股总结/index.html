八股总结

计算机网络

1.基础篇

1.1TCP/IP网络模型有几层

自顶向下分别有应用层，传输层，网络层，网络接口层，

应用层协议：HTTP、FTP、TeLent、DNS、SMTP

传输层协议：TCP和UDP

网络层协议：IP(ICMP和ARP)

网络接口层主要为网络层提供「链路级别」传输的服务，负责在以太网、WiFi 这样的底层网络上发送原始数据包，工作在网卡这个层次，使用 MAC 地址来标识网络上的设备

1.2当键入网址到显示网页，期间发生了什么？

第一步，解析url，将url拆分开，获得各部分
第二步，生产HTTP请求，根据获得的web服务器和文件名来生成请求信息
第三步，查询服务器域名对应的IP地址(这里有域名的层级关系，查询流程有迭代和递归两种方法)
第四步，将HTTP传输工作交给操作系统的协议栈

HTTP是基于TCP协议传输的，TCP提供可靠服务

TCP报文格式：源端口、目的端口，序号，确认序列，首部长度，状态位，窗口大小

在TCP传输数据前，要进行三次握手建立连接，目的是确保双方都有接受和发送的能力，保证网络连接可靠

TCP报文有长度要求，要对HTTP请求消息进行划分

MTU：IP头，TCP头加数据的最大长度，一般为1500字节

MSS：数据的最大长度，MTU - IP头 - TCP头

第五步，TCP分割数据生成TCP报文

IP包格式：版本，首部长度，服务类型，总长度，标识，标志，片偏移，TTL，协议，首部校验和，源IP地址，目的IP地址

第六步，生成IP报文

MAC头格式：接收方MAC地址，发送方MAC地址，协议类型

第七步，生成MAC报文
第八步，通过网卡将数字信息转化为电信号，末尾FCS(帧校验序列)
第九步，到达交换机，交换机根据MAC地址将信号转发到相应端口，同时通过包末尾FCS校验错误
第十步，到达路由器，通过FCS校验错误，查询路由表确定输出端口，然后发送给对方
第十一步，到达服务器对数据进行解包得到HTTP请求，然后生成HTTP响应报文发送回去。客户端接收到数据包，进行解包给HTTP进程，交给浏览器渲染页面

2.HTTP篇

2.1HTTP基本概念

HTTP是超文本传输协议

状态码：1xx：提示信息，表示目前是协议处理的中间状态，还需后续操作

​ 2xx：成功，报文已经收到并被正确处理

​ 3xx：重定向，资源位置发生变化，需要客户端重新发送请求

​ 4xx：客户端错误，请求报文有误，服务器无法处理

​ 5xx：服务器错误，服务器在处理请求时内部发生了错误

HTTP常见字段

Host：用来指定服务器的域名

Content-Length：表明回应的数据长度

Connection：常用于客户端要求服务器使用HTTP长连接，以便其它请求复用

Content-Type：用于服务器回应时，告诉客户端，本次数据是什么格式

Content-Encoding：说明数据压缩方法

GET和POST

GET的语义是从服务器获取指定的资源，请求的参数一般是写在URL中，又URL规定只能支持ASCII，所以GET请求的参数只允许ASCII字符，而且浏览器会对URL长度有限制

POST的语义是根据请求负荷(报文body)对指定资源作出处理，POST请求携带数据的位置一般在body中，数据格式可以是任意的，浏览器不会对body大小进行限制

GET和POST方法都是安全和幂等的吗？

从RFC规范定义上来说，GET方法是幂等的，而POST不是

但在实际中

  • 可以用 GET 方法实现新增或删除数据的请求，这样实现的 GET 方法自然就不是安全和幂等。
  • 可以用 POST 方法实现查询数据的请求，这样实现的 POST 方法自然就是安全和幂等
HTTP缓冲实现方式

HTTP缓冲有两种实现方式，分别是强制缓冲和协商缓冲

强制缓冲

利用相应头部中的字段实现，Cache-Control(相对时间，优先级高),Expires(绝对时间，优先级低)

协商缓冲

通过服务器端告知客户端是否可以使用缓冲的方式被称为协商缓冲，就是通过和服务器的协商结果来判断是否使用本地缓冲

实现方式：

第一种，服务器通过请求头部中的If-Modified-Since和响应头部中的last-Modified比较来实现，如果最后修改时间新，就要返回最新资源，否则就响应HTTP 304走缓冲

第二种：请求头部中的 If-None-Match 字段与响应头部中的 ETag 字段，这两个字段的意思是：

  • 响应头部中 Etag：唯一标识响应资源；
  • 请求头部中的 If-None-Match：当资源过期时，浏览器发现响应头里有 Etag，则再次向服务器发起请求时，会将请求头 If-None-Match 值设置为 Etag 的值。服务器收到请求后进行比对，如果资源没有变化返回 304，如果资源变化了返回 200。

第二种优先级更高，可以避免因为修改时间导致的错误，可以检测出的时间粒度更细

HTTP特性

HTTP/1.1

优点：简单，灵活和易于扩展，应用广泛和跨平台

缺点：无状态双刃剑(可以添加cookie来控制)，明文传输双刃剑，不安全

性能：提出了长连接的通信方式，使管道网络传输成为了可能(只解决了请求队头堵塞问题，没解决响应对头堵塞)

HTTP和HTTPS
  • HTTP 是超文本传输协议，信息是明文传输，存在安全风险的问题。HTTPS 则解决 HTTP 不安全的缺陷，在 TCP 和 HTTP 网络层之间加入了 SSL/TLS 安全协议，使得报文能够加密传输。
  • HTTP 连接建立相对简单， TCP 三次握手之后便可进行 HTTP 的报文传输。而 HTTPS 在 TCP 三次握手之后，还需进行 SSL/TLS 的握手过程，才可进入加密报文传输。
  • 两者的默认端口不一样，HTTP 默认端口号是 80，HTTPS 默认端口号是 443。
  • HTTPS 协议需要向 CA（证书权威机构）申请数字证书，来保证服务器的身份是可信的
